# PRD：技能二级细分（子能力）v2

> 状态：待确认 ｜ 作者：Claude ｜ 日期：2026-07-22
> 前置：技能选择器 v1（4 个一级技能胶囊）已上线。本 PRD 在其上增加「子能力」层。

---

## 1. 背景与目标

### 1.1 背景
当前 4 个一级技能（深度研究 / 做 PPT / 智能组卷 / 教案生成）只有一个通用入口，
用户拿到的是「万金油」输出：
- 出卷的老师想要的是「15 分钟随堂测验」，得到的却是一张 90 分钟大卷；
- 做汇报 PPT 和做课件 PPT 的结构完全不同，现在只能靠用户自己在输入框里描述。

用户要为「用途」描述付出表达成本，而大多数 C 端买家不会写 prompt——
**这正是本产品对比豆包的核心差异点：把 prompt 工程预制成可点选的成品选项。**

### 1.2 目标
1. 每个一级技能下提供 3~5 个**子能力**（场景化预设），点选即用，零 prompt 门槛；
2. 子能力以**缩略图卡片**呈现，选中即切换，输入框提示语/示例随之变化；
3. 后端按子能力注入对应的**结构预设**（题量/时长/结构/文风），产出明显更贴合场景;
4. 不改变计费模型：任何子能力 = 1 次，不引入新 SKU 维度。

### 1.3 非目标（明确不做）
- ❌ 子能力级别的差异化计费/套餐（复杂度不值得，全部含在次数内）
- ❌ 用户自定义子能力/保存模板（v3 再议）
- ❌ 子能力下再分第三级
- ❌ 真实图片缩略图（服务器出网受限 + 流量小水管；用「渐变底 + 图标」方案，见 §5）

## 3. 信息架构：能力树

**选择 = 一级技能（胶囊） + 子能力（缩略图卡片）**，二者组合决定后端预设。
每个一级技能的第一个子能力为「通用」默认项——不选也能用，行为与 v1 完全一致（兼容老用户习惯）。

### 3.1 🔍 深度研究（SSE 长报告）

| 子能力 id | 名称 | 图标 | 预设要点（注入研究计划/报告生成的风格约束） |
|---|---|---|---|
| `general` | 通用调研 | 🔍 | 默认，无额外约束（=v1 行为） |
| `finance` | 金融/行业分析 | 📈 | 报告框架：行业规模→竞争格局→产业链→风险与机会→数据表格；要求引用数据与来源；结论克制、标注不确定性 |
| `market` | 市场调研报告 | 🛒 | 框架:市场规模→用户画像→渠道→竞品对比表→SWOT→建议；商业文风 |
| `academic` | 文献综述 | 🎓 | 框架：研究脉络→流派观点对比→争议点→研究空白；学术文风、按主题组织而非罗列 |
| `policy` | 政策解读 | 📜 | 框架：政策原文要点→出台背景→影响对象→执行时间线→应对建议 |

### 3.2 📊 做 PPT（.pptx）

| 子能力 id | 名称 | 图标 | 预设要点（注入 composer 提示词） |
|---|---|---|---|
| `general` | 通用演示 | 📊 | 默认（=v1 行为） |
| `workreport` | 工作汇报 | 💼 | 结构：目标回顾→完成情况(量化)→问题分析→下步计划；每页要点式、多用数据 |
| `courseware` | 教学课件 | 🧑‍🏫 | 结构：学习目标→导入→知识讲解(分步骤、每页一个知识点)→例题→练习→小结；语言面向学生 |
| `pitch` | 产品/项目介绍 | 🚀 | 结构：痛点→方案→亮点→数据/案例→行动号召；营销文风、短句 |
| `training` | 培训分享 | 📣 | 结构：为什么学→核心概念→方法步骤→案例→练习与总结 |

### 3.3 📝 智能组卷（.docx）——预设差异最大、价值最高的模块

| 子能力 id | 名称 | 图标 | 结构预设 |
|---|---|---|---|
| `unit` | 单元测试 | 📝 | 默认。45~60 分钟，满分 100；选择+填空+解答完整题型；难度 5:3:2（基础:中档:较难） |
| `quiz` | 随堂测验 | ⏱️ | 10~15 分钟，满分 20~30；6~10 题，以选择/填空为主；难度 7:3:0，聚焦当堂知识点 |
| `homework` | 家庭作业 | 🏠 | 30~40 分钟量；题型均衡带 1~2 道提高题；难度 6:3:1；答案解析更详细（家长可对照批改） |
| `exam` | 期中/期末卷 | 🎯 | 90~120 分钟，满分 100/120；覆盖多单元、综合题压轴；难度 5:3:2；附分值分布说明 |
| `topic` | 专题训练 | 🎪 | 单知识点强化，15~20 题同型递进；难度阶梯上升；附「易错点提示」栏目 |

> 组卷预设注入位置：`EXAM_SYSTEM_PROMPT` 尾部追加「本卷用途约束」段；
> 用户输入仍可覆盖预设（用户说 20 道就 20 道，预设只兜底未指定项）。

### 3.4 📚 教案生成（.docx）

| 子能力 id | 名称 | 图标 | 预设要点 |
|---|---|---|---|
| `newlesson` | 新授课教案 | 📚 | 默认（=v1 行为）：导入→新授→巩固→小结→作业 |
| `review` | 复习课教案 | 🔁 | 结构：知识网络梳理→典型例题→易错辨析→分层练习→查漏补缺 |
| `openclass` | 公开课教案 | 🌟 | 精细版：每环节增设「设计意图」；含时间分配（精确到分钟）、师生互动设计、多媒体使用点 |
| `speech` | 说课稿 | 🎤 | 输出为说课八股：说教材→说学情→说目标→说重难点→说教法学法→说过程→说板书→说反思 |
| `hwdesign` | 作业设计 | ✏️ | 「双减」风格分层作业：基础巩固/能力提升/拓展探究三层，附设计意图与预计用时 |

> 说课稿/作业设计复用 lesson 端点与 docx 导出，仅 JSON schema 增加可选字段（见 §6）。

## 4. 交互设计

### 4.1 页面结构（桌面端）

```
┌────────────────────────────────────────────────────────┐
│  [🔍 深度研究] [📊 做PPT] [📝 智能组卷] [📚 教案生成]      │  ← 一级：胶囊（现状保留）
├────────────────────────────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐          │
│  │  📝  │ │  ⏱️  │ │  🏠  │ │  🎯  │ │  🎪  │          │  ← 二级：子能力缩略图卡片
│  │单元测试│ │随堂测验│ │家庭作业│ │期中期末│ │专题训练│          │     选中卡片高亮描边
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘          │
│   45~60分钟 · 完整题型 · 难度均衡                          │  ← 选中项的一句话说明
├────────────────────────────────────────────────────────┤
│  💡 示例：初中数学 一元二次方程 10道选择5道解答   [点击填入]  │
│  ┌──────────────────────────────────────────────┐      │
│  │ 学科/年级/知识点…（随子能力变化的 placeholder）      │      │
│  └──────────────────────────────────────────────┘      │
└────────────────────────────────────────────────────────┘
```

### 4.2 交互规则
1. **切换一级技能** → 子能力卡片行整体替换，默认选中该技能的第一个（通用/默认）子能力；
2. **点选子能力卡片** → 卡片高亮（主题色描边 + 底色加深），下方说明文字、输入框 placeholder、示例随之切换；
3. **再次点击已选中卡片**：不取消（始终有且仅有一个选中，避免「未选」状态）；
4. **示例点击** → 填入输入框（现状逻辑保留，示例文案按子能力定制）；
5. **记忆**：`(currentSkill, currentSubSkill)` 持久化到 localStorage（settings-store），下次进入恢复；
6. **对话历史/结果卡**：文件结果卡上显示「📝 智能组卷 · 随堂测验」二级面包屑，让用户知道当次用的是什么预设；
7. **默认路径零成本**：用户不碰子能力行为与 v1 完全相同——新功能纯增益，无学习成本。

### 4.3 移动端（≥90% 用户）
- 子能力卡片行**横向滚动**（overflow-x, 隐藏滚动条, 右缘渐隐提示可滑动）；
- 卡片尺寸 72×72（图标 28px + 名称 11px 单行），触控热区 ≥44px；
- 选中项说明文字折行显示在卡片行下方，不挤压输入区；
- 一级胶囊 + 二级卡片总高度控制在 ~140px 内，保证输入框和键盘弹起后仍可见。

### 4.4 状态流转

```
进入聊天页
   │ 读 localStorage（无则 research/general）
   ▼
[一级+二级 已选] ──点一级──► 二级重置为该技能默认项
   │  ▲                        │
   点二级 ─────────────────────┘
   ▼
输入/点示例 → 提交
   │ research: SSE 流（携带 sub_skill）
   │ 文件类: POST（携带 sub_skill）→ loading 文案含子能力名 → 结果卡（二级面包屑+下载）
   ▼
失败 → 「本次失败，未扣除次数」（现状保留）
```

### 4.5 文案规范
- 子能力名 ≤4 字（卡片放得下）；说明一句话 ≤20 字，说人话（「10~15分钟小卷」而不是「轻量化测评方案」）；
- 每个子能力 2 条示例，**示例必须带版本+年级**（配合教材范围约束，如「人教版九上数学…」）。

## 5. 视觉：缩略图卡片规范

不用真实图片（出网受限 + 流量成本），用**代码生成的轻量卡片**：

- 卡片 = 圆角 12px 容器，**每个一级技能一种主题色渐变底**（研究-蓝、PPT-橙、组卷-绿、教案-紫），同技能下子能力共用色系、靠图标区分 → 视觉上强化「同属一个模块」；
- 图标 = emoji（28px，居中）。零资源文件、零请求，深浅色主题自适应；
- 选中态：2px 主题色描边 + 背景饱和度提升 + 图标轻微放大（transform: scale(1.08)，120ms）；
- 未选态：低饱和底 + 60% 不透明度文字；
- 暗色模式：渐变底加深，描边亮度提升。

> 若后续想升级成插画缩略图：本地 SVG 打包进前端即可，不引入外链。本期不做。

## 6. 后端设计

### 6.1 原则
子能力 = **纯提示词预设注入**，不新增 graph、不新增端点、不动计费。
预设集中放在一个文件，运营期改文案不动业务代码。

### 6.2 改动清单

| 文件 | 改动 |
|---|---|
| `src/skills/presets.py`（新增） | `SUB_SKILL_PRESETS: dict[skill][sub_id] -> 约束文本`；`get_preset(skill, sub_id)` 未知 id 回退 default（前后端版本不同步时安全降级） |
| `src/server/app.py` | 4 个入口接收可选 `sub_skill: str`；exam/lesson/ppt 在调用生成函数前把预设文本拼进 prompt 链；research 把预设注入 planner 的补充指令 |
| `src/skills/exam.py` | `generate_exam(prompt, config, preset_text="")`：system prompt 尾部追加「本卷用途约束」段 |
| `src/skills/lesson.py` | 同上；`speech`/`hwdesign` 两个子能力使用**变体 JSON schema**（说课八股/分层作业字段），docx 导出对应增加两个渲染分支 |
| `src/skills/docx_export.py` | 新增 `build_speech_docx`、`build_hwdesign_docx`（复用现有样式函数） |
| `src/ppt/graph/*` | composer 提示词模板追加 preset 段 |
| runs 表 | `skill` 字段值扩展为 `"exam:quiz"` 形式（`skill:sub` 复合串，不加列、不迁移） |

### 6.3 API 变更（向后兼容）

```
POST /api/exam/generate    {prompt, sub_skill?: "unit|quiz|homework|exam|topic"}
POST /api/lesson/generate  {prompt, sub_skill?: "newlesson|review|openclass|speech|hwdesign"}
POST /api/ppt/generate     {content, sub_skill?: "general|workreport|courseware|pitch|training"}
POST /api/chat/stream      {..., sub_skill?: "general|finance|market|academic|policy"}
```
`sub_skill` 缺省/未知 → default 预设，等价 v1 行为。

### 6.4 与教材范围约束的关系（并行工作流）
组卷/教案的预设文本中统一包含一句：「严格限定在用户指定学段的课标范围内，
解法不得使用更高学段知识；未指明教材版本时按人教版最新课标」。
教材目录 JSON（ChinaTextbook 提炼，进行中）落地后，在同一注入点追加对应年级的单元目录切片——
两个工作流互不阻塞。

### 6.5 前端改动清单

| 文件 | 改动 |
|---|---|
| `web/src/core/skills.ts` | `SkillDef` 增加 `subSkills: SubSkillDef[]`（id/名称/emoji/说明/placeholder/示例）；导出 `getSubSkill()` |
| `web/src/core/store/settings-store.ts` | 增加 `currentSubSkill`，随 skill 切换重置为默认；持久化 |
| 技能选择器组件 | 胶囊下方渲染子能力卡片行（横滚 + 选中态） |
| `web/src/core/api/generate.ts` / SSE 请求 | 请求体带 `sub_skill` |
| 结果卡组件 | 显示「一级 · 二级」面包屑 |

## 7. 数据与运营

- runs 表 `skill:sub` 复合值天然支持按子能力统计：哪个子能力用得多 → 决定下一步做深哪个（如随堂测验用量高 → 优先给它上教材目录约束）；
- 闲鱼商品详情页文案可直接列子能力清单（20 个场景），显著增强「功能丰富」感知；
- 上新子能力 = presets.py 加一条 + skills.ts 加一条，无需数据库/部署以外操作。

## 8. 验收标准

1. 4 个一级技能各有 3~5 个子能力卡片，切换流畅，选中态明确；
2. 移动端（375px 宽）卡片行可横滑、无布局破损，输入框不被挤出首屏；
3. 同一句输入「初中数学 一元二次方程」在 随堂测验/期中期末 两个子能力下，产出的试卷在题量、时长、难度分布上有肉眼可见差异；
4. 说课稿、作业设计产出 Word 结构符合各自八股/分层格式；
5. 不带 `sub_skill` 的老请求行为与 v1 完全一致；
6. 每次产出扣费仍为 1 次；失败不扣（现状回归不被破坏）。
